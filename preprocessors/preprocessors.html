Установка

Лучше и удобнее скачать приложение - рекомендовано Koala, оно бесплатное и простое

Скачиваем приложение, устанавливаем, ддобавляем папку нашего проекта, а в проекте создаем папку css и файл style.sass

Нажимаем на поиск, нажимаем на файл и Ставим галочки на Auto Compile, Source Map, Autoprefix.

Подключаем файл style.css к нашему документу

после этого все пишем в файле style.sass, а оно будет с помощью koala компелироваться в css

Sass

Sass (от англ. Syntactically Awesome Stylesheets — синтаксически замечательные таблицы стилей) — мета-язык описания стилей.

Для работы требуется утилита, которая (на этапе разработки) код на языке Sass транслирует в CSS-файл; либо серверный модуль, который при отдаче Sass-кода клиенту «на лету» транслирует его в CSS-код.

sdfghjl;l
  1. Вместо фигурных скобок, для обозначения вложенности синтаксических конструкций используются отступы. Нет точек с запятой после стилевых описаний, поэтому, записывается только одно стилевое описание в строке.
  2. Вместо иерархических селекторов указываются селекторы с отступом.
  3. Отступы также можно использовать для сокращения записи длинных CSS-свойств.
  4. Чтобы не повторять значения стилевых свойств, можно использовать переменные, и вычислять значения выражений с переменными.
    НА картинке ошибка. Правильно не через равно задавать, а через ДВОЕТОЧИЕ
  5. Чтобы не повторять целые куски стилевых описаний, можно использовать миксины.
  6. Миксин может получать аргумент.

Scss

SCSS (Sassy CSS) — мета-язык описания стилей, наследник Sass. Ориентирован на то, чтобы быть синтаксически ближе к CSS.

Для работы требуется серверный модуль (препроцессор SCSS), который при отдаче SCSS-кода клиенту «на лету» транслирует его в CSS-код. Для LESS также существуют серверные модули, но более удобно использовать JavaScript-модуль, который на стороне клиента программно вносит нужные изменения в стилевые таблицы.

Фигурные скобки работают, как и в css и точки с запятой тоже. Но классы можно писать, как и в sass, а также не важно, сколько отступов сделали

  1. В отличие от Sass, фигурные скобки и точки с запятой используются так же как и в CSS. Поэтому, в простых случаях код SCSS вообще не отличается от кода CSS.
  2. Иерархические селекторы тоже поддерживаются, но синтаксис немного другой.
  3. Также можно использовать символ «&» для ссылки на родительский селектор.
  4. Можно использовать переменные, и вычислять значения выражений с переменными, точно так же как в Sass, только синтаксис немного другой.
  5. Можно использовать миксины (в LESS другой синтаксис).
  6. В миксинах можно использовать @content.
  7. Можно склеивать строки.
  8. Можно использовать циклы (LESS не поддерживает).
  9. Можно импортировать (вклеивать) другие SCSS-файлы. При этом препроцессор SCSS отдаст клиенту (браузеру) один, объединённый CSS-файл.
