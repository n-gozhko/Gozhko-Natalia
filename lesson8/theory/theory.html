Формы и позиционирование

Атрибут Action - это url-адрес, который необходим для того, чтобы мы могли отправить что-то на сервер. Т.е. поэ тому адресу лежит обработчик и там back-end разработчик обрабатывает данные, которые прилетели. Т.е. мы ссылаемся на этот url, и он обрабатывает нам ту информацию, которую человек на сайте передал через форму, т.е. input.
В качестве примера можно брать сейчас ссылку: http://fe.it-academy.by/TestForm.php

Input - кнопка. У нее есть много разных видов. Text - текст, submit-отправить информацию на url-адрес.

В теге input нужно всегда указывать name. Это необходимо для того, чтобы бэк-разработчик понимал, что за данные ему приходят. С помощью метода get он получает ссылку, в которой указаны данные, которые он получил:
http://fe.it-academy.by/TestForm.php?fio=Natalia&age=24&otpravka=send Т.е. пользователь написал ФИО - Natalia, а возраст - 24.

По умолчанию кнопка называется *отправить*. Чтобы изменить название, задать атрибут value

Рассмотрим, какие есть Headers при формировании страницы после обработки полученных данных при МЕТОДЕ GET

  • General - общие. Здесь прописывается url-адрес. Статус о том, что у нас прошло всё хорошо.
    Если что-то неверно, то указывается, какая ошибка, например 404, 401 или др. Это необходимо для того, чтобы понять что пошло не так, что не работает. Также есть удалённый адрес, т.е. IP-адрес.
  • Response Headers - ответы. Включено то, что у нас сейчас постоянное соединени.
    Также какой контент, лучше архив.
    Длина контента - это длина html страницы, которая сформировалась после обработки ответов.
    Тип отправляемого контента и дата отправки.
    Тайм-аут соединения, например каждые 20 сек проверяет соединение с интернетом.
    Сервер - самые популярных два - это апатч и эджинз (nginx).
    Как операционная система.
  • Request Headers - запросы, вопросы. Тут должны быть прописаны, какие файлы может отправлять, каким форматом сжатия. Гораздо проще заархивировать, и после получения разархивировать, чем наобоорт.
    Доступные языки также есть,
    Cookie-файлы - это какие-то локальные данные, которые сохраняются в браузере и потом могут передаваться на сервер.
    Есть также host - доменное имя, это куда мы стучались.
    Реферальная ссылка - это откуда мы изначально пошли, т.е. это и есть ссылка нашей папки, где лежит фай локально.
    User-agent - это необходимо для кроссбраузерности. Показывает, с какого браузера и версии заходит к нам на сайт человек. Но инфа более ненужная. Непонятно, как из нее инфу обработать.
  • Query String Parameters - праметры строкового запроса. Включаются те параметры, которые мы получили. Т.е. параметры: фио, возраст и т.д.

МЕТОД POST

После ввода данных, в отличии от метода get, в методе пост отображается адресная строка без полученных данных: С одной стороны кажется, что это безопасно, но на самом деле нет. Все равно мы можем открыть network и все эти параметры увидеть.
В этом и есть вся разница. В методе POST данные видны только в headers данных, т.е. form date - тело запроса.

Метод get применяется в таких сайтах, как например онлайнер. Кодга мы выбираем сбоку параметры, например год машины, вид топлива, марку и т.д. у нас в адресной строке все эти параметры отображаются. Когда мы отправляем кому-то ссылку, он сразу попадает на страницу по этому запросу с отобранными параметрами.
При методе POST человек пройдя по ссылке попадет на общую страницу со всеми машинами, без выбранных нами параметров.

Атрибут textarea используется, чтобы было поле для ввода текста и мы видели, что вводили.

Некоторые возможные значения атрибута type:

  1. text — поле ввода текста.
    • Атрибут value задаёт начальное значение текста.
    • Атрибут maxlength задаёт максимальное количество символов, которые можно ввести в поле.
    • Атрибут readonly (без значения) делает поле доступным только для чтения (значение в поле нельзя изменить); на внешний вид элемента этот атрибут не влияет.
    • Атрибут placeholder позволяет задать текст, который будет появляться в незаполненом элементе ввода текста как подсказка о том, что именно нужно вводить в поле (этот атрибут поддерживается не везде — Chrome, Firefox, Opera 11.5+, IE 10+).
  2. password — поле ввода пароля (вводимые символы отображаются звёздочками).
    • Атрибут maxlength задаёт максимальное количество символов, которые можно ввести в поле.
  3. checkbox — птичка (может быть установленной либо сброшенной).
    • Атрибут checked (без значения) задаёт начальное состояние птички.
    • Атрибут value задаёт значение, которое будет отправлено на сервер при установке птички; по умолчанию это значение 'on'.
  4. radio — поле для выбора одного из нескольких вариантов
    • Каждый из вариантов должен иметь одно и то же значение name, и разные значения value.
    • Атрибут checked (без значения) указывает, какой из вариантов должен выбран изначально).
  5. file — поле для выбора файла с компьютера пользователя для отправки на сервер.
    • Для правильной работы элемента в теге form обязательно должны быть указан атрибуты method=post.
    • Для правильной работы элемента в теге form обязательно должны быть указан enctype="multipart/form-data".
  6. hidden — невидимый элемент. Предназначен для отправки на сервер дополнительных данных, которые требуются скриптом-обработчиком и известны разработчику веб-страницы, но не должны вводиться пользователем напрямую.
    • Атрибут value задаёт значение элемента.
  7. submit — кнопка отправки формы (т.е. всех элементов и их значений) серверному скрипту.
    • Атрибут value задаёт подпись на кнопке.
  8. reset — кнопка сброса формы (т.е. значения всех элементов возвращаются к тому состоянию, которое было при открытии формы).
    • Атрибут value задаёт подпись на кнопке.
  9. button — кнопка без предзаданного поведения (часто используется для вызова JavaScript).
    • Атрибут value задаёт подпись на кнопке.

Например



В типе checkbox по умолчанию, если мы отправляем поставленную галочку - то пишется значение on. Если пустую галочку - то значение не прописывается. Но можно переименовать его через атрибут value

Если мы задаем какое-то поле с одинаковым названием, например 5 галочек с одним названием check, то при получении данных, бэк-разработчик не поймёт, на какую именно галочку человек выбрал. В этом случе, можно:

  • Задать каждой галочке своё name. Но в этом случае бэк-разработчик получает много строчек отдельно по каждому ответу.
  • Можно задать массив, т.е. для все галочек поставить имя одно - check, а уже в value прописать разные название. Тогда разработчику придёт все в одной строчке, но он сможет увидеть, к какому варианту принадлежит ответ.

Если необходимо выбрать только одно значение - используется radio-кнопка. В этом случае уже массив не нужен.

Благодаря name мы соединяем радиокнопки. Если мы пропишем их много, но везде одно название - то из всех кнопок можно выбрать только одну.

File - загрузка файлов. Для того, чтобы все хорошо работало, необходимо использовать метод post, а также в теле form прописать enctype="multipart/form-data"

Hidden необходим например при голосовалках, когда у нас абсолютно одинаковые формы, но разработчик нужно понять, на какую форму человек ответил.
Т.е. на странице эта строка не отображается, но бэк-разработчик увидит название формы


  • Атрибут autofocus тега input (без значения) автоматически фокусирует данный элемент формы при загрузке страницы (не работает в Internet Explorer до версии 10).
  • Атрибут disabled тега input (без значения) делает элемент формы недоступным. Обычно такие элементы формы имеют серый цвет текста и/или фона. Этот атрибут можно использовать с любым типом элемента input. Значения из недоступных полей формы не передаются на сервер.

Textarea - создает поле ввода текста, как отдельный тег .Его можно ресайзить, но если мы хотим, чтобы пользовать не мог менять размер поля - задаем в стиле resize: none.
Текст будет отображаться у бэка тоочно также, как ввел пользователь: со всеми пробелами, и прочим.

Select также отдельный тег - выпадающий список. Должен содержать элементы option и, при необходимости, optgroup.



  • Атрибут multiple (без значения) позволяет пользователю выбирать сразу несколько позиций из списка; при этом список перестаёт быть выпадающим.
  • Атрибут size позволяет установить количество отображаемых в списке строк, если список невыпадающий (т.е. при наличии атрибута multiple).

Чтобы корректно приходила информация бэк-разработчику, необходимо добавить массив.

Тег optgroup представляет собой контейнер, внутри которого располагаются теги option объединенные в одну группу. Особенностью тега optgroup является то, что он не выделяется как обычный элемент списка, акцентируется с помощью жирного начертания, а все элементы, входящие в этот контейнер, смещаются вправо от своего исходного положения.
С помощью атрибута Lable - задается текст, который будет входить в список в виде заголовка группы.

Атрибут disabled (без значения) делает элемент списка недоступным для выбора.

Парный тег button позволяет расположить на форме такую кнопку, внутри которой можно разместить произвольный HTML-код.

Атрибут type тега button задаёт поведение кнопки:

  • submit — отправка формы (т.е. всех элементов и их значений) серверному скрипту.
  • reset — сброс формы (т.е. значения всех элементов возвращаются к тому состоянию, которое было при открытии формы).
  • button — кнопка без предзаданного поведения;

Атрибут disabled (без значения) запрещает нажатие кнопки.
